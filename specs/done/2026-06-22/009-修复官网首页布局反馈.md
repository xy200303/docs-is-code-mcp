用户反馈当前官网首页很丑，并质疑首页为什么还有侧边栏。需要修正 VitePress 首页，使其使用真正首页布局而不是文档页侧栏布局；视觉上去掉厚重、模板化、泛 SaaS 的首页结构，保留真实项目入口和命令内容；验证桌面和移动端无侧边栏、无串项目、无水平溢出或裁切。

1. 首页布局
  - 条件：访问 VitePress 官网首页 /
  - 触发入口：浏览器请求 http://127.0.0.1:4174/
  - 输入与前置状态：docs/index.md front matter 使用 layout: home，页面加载 /site.css
  - 执行过程：
    1. VitePress 按 home layout 渲染首页
    2. 自定义 home-shell 内容展示项目名称、安装命令、工作流和真实入口
    3. site.css 只在 .VPContent.is-home 下隐藏 VitePress sidebar/local nav
  - 输出结果：首页无文档侧边栏，保留顶部导航和首页入口内容。
  - 副作用：只改变静态文档站首页渲染，不影响 MCP 服务运行。
  - 默认行为：普通文档页仍使用 VitePress 默认文档布局和 sidebar。
  - 边界处理：未记录
  - 结果摘要：首页不再使用 layout: page，不出现文档页侧栏。
  - 验证：docs/index.md front matter、test/unit.ts 回归断言、桌面/移动截图检查
  - 关联文件：
    - `docs/index.md`
    - `docs/public/site.css`
    - `test/unit.ts`

2. 移动端宽度
  - 条件：390px 移动视口访问首页
  - 触发入口：Edge DevTools Protocol 设置 width=390 height=900 mobile=true 后访问首页
  - 输入与前置状态：页面包含较长中英文说明和 npm/specc 命令
  - 执行过程：
    1. site.css 重置 .VPContent.is-home、.VPHome、.vp-doc.container 的首页宽度
    2. home-shell 使用视口内宽度和 16px 左右边距
    3. 命令行按 token 渲染并允许 anywhere/break-all 换行
    4. CDP 读取 documentElement 和 body 的 clientWidth/scrollWidth
  - 输出结果：首页内容在 390px 视口内完整显示，命令换行，未产生水平滚动。
  - 副作用：无运行时数据写入；只生成本地 .tmp/screens 截图文件用于验收。
  - 默认行为：未进入移动断点时继续使用桌面两栏 hero 和四列工作流。
  - 边界处理：未记录
  - 结果摘要：CDP 指标为 clientWidth=390、scrollWidth=390、body scrollWidth=390；截图 .tmp/screens/home-mobile-cdp.png 无裁切。
  - 验证：Edge DevTools Protocol mobile screenshot and width metrics
  - 关联文件：
    - `docs/public/site.css`
    - `docs/index.md`
